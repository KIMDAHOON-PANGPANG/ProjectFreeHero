## 🔒 고정된 핵심 설계 결정 (P0 GDD #1)

이 결정들은 이미 GDD에 기록됨. 바꾸려면 GDD를 먼저 수정할 것.

### Auto-Target + Warp 시스템
- **락온 방식**: 스티키 락 (탭마다 재탐색 X, 한 번 락되면 유지)
- **락 해제 조건**: 타겟 사망 / 거리 > `MaxLockDistance` (1500uu) / `LockIdleTimeout` (2.0s) 경과
- **타겟 스코어**: `Score = Distance + AngleFromFacing × ε` (ε=0.001) — **거리 지배, 각도는 타이브레이커만**
- **워프 거리**: `MaxWarpDistance` = 1000uu
- **워프 지속**: `WarpDuration` = 0.08s, **EaseOutQuad**
- **워프 시각화**: 리플레이스드 스타일 루트 이동 (텔레포트 아님)
- **엘레멘트 보정**: Electric +30% 사거리, Ice/Fire 없음
- **공격 대상 수**: `MaxAttackTargets` = **1** (DataAsset에서 튜닝 가능)
- **차단 규칙**: 벽 뒤 차단, 낭떠러지 차단 (횡스크롤 안전)
- **홀드 규칙**: 넉백 중에도 락 유지, 재탐색 안 함
- **Group Token**: 타겟 선택과 **독립** (공격 권한 풀은 별개 시스템)

### 구현 메모 (GDD 하단에 기록됨)
- `BP_AutoTargetComponent` — 컴포넌트로 캐릭터에 부착
- `DA_AutoTargetConfig` — DataAsset 튜닝 값
- `BPI_Targetable` — 타겟 인터페이스
- `BP_WarpMoveTask` — 루트 이동 BlueprintAsyncTask

### 수락 기준 합계
