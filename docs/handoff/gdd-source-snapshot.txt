[Heading 1] 2. 세계관 & 내러티브

[Heading 2] 2.1 월드 설정
[Normal] 초능력이 존재하는 현대 한국. 3개 범죄 조직이 감염원을 유통하고, 감염된 시민들이 거리에서 폭주. 히어로와 사이드킥이 공존하는 세계.
[Heading 2] 2.2 주인공
[Normal] 초능력 없는 체술형 사이드킥. 히어로에게 서포트 기어를 지급받아 속성 전투가 가능. 인게임에서 캐릭터 스프라이트는 동일, 속성 차이는 버프 이펙트 + 히트 이펙트 색상 + 스킬 연출로만 표현.
[Heading 2] 2.3 스토리 기둥
[Normal] 성장: "나도 히어로가 되겠다" — 플레이어의 감정 드라이브.
[Normal] 부조리: 사이드킥이 느끼는 체계적 불공정을 플레이 과정에서 체감.
[Normal] 부패: 부패한 히어로가 감염원 유통에 관여하고 있다는 진실이 런 진행 중 드러남.
[Heading 2] 2.4 내러티브 페이오프
[Normal] 부패한 히어로 대릴 시 기어를 뮯김 → 영구 해금한 체술 스킬만으로 전투 → 승리 = 진정한 히어로가 된 순간.

[Heading 1] 3. 전투 시스템

[Heading 2] 3.1 조작 (4입력)
[TABLE_START]
입력 | 액션 | 상세
탭 | 광클 공격 | 빠른 연타, 적 사이를 워핑하며 자동 연계. 프리플로우의 핵심 루프.
홀드 | 넍백 | 적을 날려서 다른 적/벽에 충돌. 흐름 중 수시로 사용, 피니시/막타 전용이 아님.
패링 | 타이밍 패링 | 노란색 텔레그래프에 타이밍 입력. 성공 시 큰 경직 + 카운터 창.
회피 | 저스트 회피 | 붉은색 텔레그래프에 저스트 회피. 패링 불가능한 공격에만 사용.
[TABLE_END]

[Heading 2] 3.2 프리플로우 슬롯 (6개 패시브 부착점)
[Normal] 모든 빌드에서 6개 슬롯 전부 사용 가능. 히어로 속성에 따라 특정 슬롯의 보상이 커질 뿐, 다른 슬롯을 제거하지 않음.
[TABLE_START]
슬롯 | 설명 | 구현 포인트
광클(Rush) | 탭 연타 공격. 주력 데미지 소스. | ComboSystem, HitCount, DamageMultiplier
패링(Parry) | 노란색 텔레그래프 공격에 대한 카운터. | ParryWindow, ParryResult SO
회피(Dodge) | 붉은색 언블록커블 공격에 대한 회피. | DodgeWindow, iFrame, DodgeResult SO
넍백(Knockback) | 홀드 공격으로 적 발사. 환경 상호작용 트리거. | KnockbackForce, LaunchDirection, CollisionCheck
이동(Warp) | 적→적 자동 이동. 거리/속도 보너스 변수. | AutoTargeting, WarpSpeed, WarpDistance
피니시(Finish) | 적 처치 순간. 처치 시 발동 효과. | OnKillEvent, FinishAnimation, OnDeathEffect SO
[TABLE_END]

[Heading 2] 3.3 스타일 랭크 시스템
[Normal] 전 속성 공통. 저스트 타이밍 연계 + 피격 미발생 시 랭크 상승. S랭크 달성 시 히어로 어시스트 발동 가능.
[TABLE_START]
랭크 | 전기 | 파이어 | 아이스
D→C | 속도 +10% | DoT 데미지 +15% | 패링 윈도우 +10%
C→B | BGM 레이어 2 추가 | 폭발 범위 +20% | 동결 지속 +0.5s
B→A | BGM 레이어 3, 히트 수 +1 | 화면 불꽃 VFX 강화 | 화면 색온도 냉각 시프트
A→S | 순간이동 연속 처형 연출 | 풀 인페르노 모드 | 전장 범위 동결
[TABLE_END]

[Heading 2] 3.4 연출 에스컬레이션 (포스트 프로세싱)
[Normal] 색온도 시프트: 랭크 상승 → 배경 채도 감소 + 속성 색 티트 화면 전체 적용. S랭크에서 캐릭터 외곽 속성 색 림라이트. 셸이더 파라미터 하나로 제어.
[Normal] 스피드라인 + 잔상: B랭크부터 워핑 시 1프레임 잔상 생성(속성 색). S랭크에서 2~3프레임. 스프라이트 복제 + 알파 감소로 구현.
[Heading 2] 3.5 구현 우선순위 정리
[TABLE_START]
P | 시스템 | 핵심 컴포넌트 | 비고
P0 | 자동 타겟 워핑 | AutoTargetSystem | 프리플로우 존재 이유
P0 | 탭/홀드 공격 분기 | CombatInputHandler | 4입력 체계의 기반
P0 | 캐릭터 스테이트 머신 | PlayerStateMachine | Idle/Attack/Parry/Dodge/Knockback/Hit/Warp
P0 | 히트 리액션 | HitReactionSystem | Flinch/Stagger/Knockback/Launch

[Heading 1] 4. 히어로 시스템 (빌드 테마)

[Normal] 히어로는 플레이어블이 아님. 서포트 기어(속성 테마) + 어시스트(짧은 시네마틱 공격)를 제공하는 존재.
[Normal] 히어로당 필요 리소스: 선택 화면 일러스트 + 어시스트 모션 1개 + 대사 + SO 데이터 세트.

[Heading 3] 4.1 전기 히어로
[TABLE_START]
빌드 방향 | 광클 강화. 히트 수 증가, 연속 히트 보너스, 체인 라이트닝.
핵심 메커니즘 | 광클로 감전 스택 축적 → 충전 완료 시 방전(범위 한방).
오디오 특징 | BGM 레이어링: 랭크 D=드럼, C=베이스, B=신스, A=풀트랙.
S랭크 연출 | 순간이동 연속 처형: 화면 내 모든 적을 순간 이동하며 연속 타격.
어시스트 | 화면 전체 번개 낙뢰.
[TABLE_END]

[Heading 3] 4.2 파이어 히어로
[TABLE_START]
빌드 방향 | 넍백 강화. 도트댐 + 적 충돌 시 폭발 + 도트 전이.
핵심 메커니즘 | 도트 건 적을 밀집 지역으로 날려서 연쇄 폭발. 볼링 플레이.
S랭크 연출 | 풀 인페르노: 화면 불바다, DoT 2배.
어시스트 | 거대 화염 펀치로 적 무리 날림.
[TABLE_END]

[Heading 3] 4.3 아이스 히어로
[TABLE_START]
빌드 방향 | 패링 강화. 패링 시 동결 + 동결 적 타격 시 파편 범위 피해.
핵심 메커니즘 | 적 공격 유도 → 패링 카운터 → 동결 → 파편 폭발.
S랭크 연출 | 전장 범위 동결: 모든 적 동결, 사이드킥이 직접 부숨.
어시스트 | 범위 동결 파동 → 사이드킥이 후속 타격.
[TABLE_END]

[Heading 3] 4.4 구현 구조
[Normal] SO_HeroData: heroID, elementType, basePassives[], assistAnimClip, assistDamageData, rankRewards[], bgmLayers[]
[Normal] 설계 원칙: 모든 패시브는 "조건 충족 시 보너스"이지 "다른 행동 제거"가 아님.

[Heading 1] 5. 뫀스터 설계

[Heading 2] 5.1 세계관 근거
[Normal] 범죄 조직이 감염원 유통 → 감염된 시민 = 잡몹 → 조직 간부 = 중보스/보스. 감염 진행도 = 위협도 = 외형 변이도.
[Heading 2] 5.2 4대 기본 타입
[TABLE_START]
타입 | 강제 액션 | 외형 | 구현 포인트
일반 감염자 | 광클(연료) | 후드 시민, 눈 발광 | 구성비 60~70%. 느린 접근. SO: EnemyData_Standard
텔레그래프형 | 패링 유도 | 한쪽 팔 비대, 결정체 | 노란색 번짝 후 공격. SO: EnemyData_Telegraph
돌진형 | 회피 강제 | 하체 변이, 낮고 빠른 실루엿 | 붉은색 잔상 트레일. 패링 불가. SO: EnemyData_Charge
아머형 | 넍백 강제 | 몸 전체 결정/껍질 | 일반 공격 경직 안 먹힘. 다른 적 날려서 충돌시키거나 벽으로 밀어야 아머 파괴. 아머 깨지면 안에서 인간 형태 드러남. SO: EnemyData_Armor
[TABLE_END]

[Heading 2] 5.3 보스 (조직 간부)
[Normal] 자발적 감염자. 정장 위에 속성 갑옷 레이어. 인간 형태 유지. 3페이즈 전투:
[Normal] Phase 1: 텔레그래프 위주 (패링 집중). 인간 외형.
[Normal] Phase 2: 아머 활성화 + 잡몹 소환. 잡몹을 보스에게 날려서 아머 파괴.
[Normal] Phase 3: 돌진 패턴 연속 (회피 집중). 근접 괴물화. 인간성 상실.
[Heading 2] 5.4 에셋 효율
[Normal] 적 스프라이트 4세트 × 3색 팔레트 스왕(조직별) = 체감상 12종. 실제 제작은 4세트. 보스는 고유 스프라이트 (총 3체 for v1).

[Heading 1] 6. 3대 범죄 조직 (진영 테마)

[Normal] 기본 4타입 적은 공유. 조직별 배치 비율 + 행동 템포 + 환경 위험 + 배경 톤으로 차별화.
[TABLE_START]
 | 전기 조직 | 파이어 조직 | 아이스 조직
범죄 유형 | 해킹/정보 범죄 | 무기 밀매/폭력 | 마약 유통/지능 범죄
적 특성 | 이동속도 빠름, 돌진형 비율 높음 | 아머형 비율 높음, 덩치 큰 적 | 텔레그래프형 비율 높음, 교대 공격
전투 체감 | 반응 속도 싸움, 빠른 템포 | 느리지만 한 방 아픔, 공간 압박 | 차분하지만 정밀, 읽기 싸움
환경 | 네온 골목, 서버룸, 클럽 | 공장, 창고, 공사현장 | 연구소, 항구, 냉동창고
배경 톤 | 파란/보라 네온 | 붉은/주황 산업톤 | 청백/냉회색
환경 위험 | 전기 패널, 반사형 벽 | 폭발물, 화염 장판 | 냉각 장치, 미끄러운 바닥
[TABLE_END]

[Normal] 전략적 조합: 히어로 선택 × 조직 조합에 따라 공략 루트 변화. 같은 스테이지를 다른 히어로로 가면 전투 방식이 달라짐.

[Heading 1] 7. 환경 상호작용

[Normal] 모든 환경 상호작용은 넍백 결과로 자연스럽게 발동. 별도 상호작용 키 없음. "적을 어디로 날리느냐"가 곧 환경 활용.
[Heading 2] 7.1 공통 컴포넌트 설계
[Normal] EnvironmentObject 컴포넌트: bool isOneShot 하나로 분기. SO_EnvironmentData에 데미지/범위/이펙트/지속시간 정의.
[TABLE_START]
속성 | isOneShot = true | isOneShot = false
동작 | 한 번 발동 후 파괴 | 유지되며 반복 판정
예시 | 폭발물, 파괴형 벽, 냉각 장치 | 반사형 벽, 감전 바닥, 얼음 패치
역할 | 스테이지당 소비 자원 | 지속적 지형 전술
[TABLE_END]

[Heading 2] 7.2 환경 오브젝트 목록
[TABLE_START]
오브젝트 | 타입 | 효과
파괴형 벽 | One-Shot | 적 충돌 시 파괴 + 추가 데미지. 다음 구역 개방 가능.
반사형 벽 | Persistent | 적이 벽에 부딪히면 틀겨나오며 경직. 핑퉁 콤보.
폭발물 | One-Shot | AoE 폭발. 파이어 빌드와 DoT 전이 시너지.
전기 패널 | One-Shot | 파괴 시 감전 장판 생성. 전기 스택 축적 가속.
냉각 장치 | One-Shot | 냉기 영역 생성. 적 감속 + 동결 확률 증가.
난간/단차 | Persistent | 적을 아래층으로 떨어뜨림. 낙하 데미지.
[TABLE_END]

[Heading 1] 8. 로그라이크 구조

[Heading 2] 8.1 런 플로우
[Normal] 쉘터 → 히어로 선택(기어 수령) → 스테이지 진입 → 전투 → 보상 선택지(2~3개) → 다음 스테이지 루트 선택(2갈래, 적 조합/보상 미리보기) → 반복 → 사망 시 쉘터 복귀
[Heading 2] 8.2 스테이지 타입
[Normal] 웨이브형: 전투 돌입 → 웨이브 진행 → 전부 종료 → 클리어
[Normal] 판단형: 전투 돌입 → 배치된 적 소거 → 클리어
[Normal] 보스형: 보스 전투 돌입 → 3페이즈 → 보스 사망 → 클리어
[Heading 2] 8.3 빌딩 3층 구조
[Normal] 1층 — 히어로 선택 (런 방향): 속성 테마 + 어시스트 결정. SO_HeroData 교체.
[Normal] 2층 — 런 내 패시브/액티브 (빌드 심화): 스테이지 클리어 보상으로 획득. 패시브 풀 15~20개. SO_PassiveData[].
[Normal] 3층 — 리스크-리워드 (극단적 방향): 트레이드오프 선택지. SO_RiskRewardData.
[Heading 2] 8.4 영구 성장 (메타 진행)
[Normal] 나인솔즈 방식. 기어(속성)와 무관한 사이드킥 본체 체술 성장.
[Normal] 초반: 기본 공격 + 패링
[Normal] 중반: 저스트 회피 해금, 환경 상호작용 해금
[Normal] 후반: 고급 체술 / 초능력 계열
[Normal] 설계 원칙: 모든 해금 스킬은 프리플로우 흐름 안에서 끄김 없이 사용 가능해야 함.

[Heading 1] 9. 리플레이 설계

[Normal] 핵심 원칙: 전투 문법을 늘리지 말고, 같은 문법이 쓰이는 맥락을 다양하게 만들 것.
[TABLE_START]
벡터 | 전투 체감 변화 방식
적 구성 비율 | 조직별 4타입 비율 변화. 같은 조작, 다른 퍼즐.
환경 변수 | 좁은 복도 vs 넓은 옥상, 단차, 위험 오브젝트.
미션 조건 | 시간 제한, 노피격, 처치 순서. 코드 몇 줄 수준.
히어로별 리듬 차이 | 같은 스테이지, 다른 히어로 = 다른 최적 플레이.
패시브 조합 시너지 | 15~20개 패시브에서 2~3개 조합 시 뜨는 "사기" 빌드.
스타일 랭크 관리 | 어시스트 타이밍을 전략적 자원으로 운용.
[TABLE_END]

[Heading 1] 10. 비주얼 & 오디오 전략

[Heading 2] 10.1 아트 파이프라인
[Normal] 3D 모델 → 픽셀 셸이더 → 2D 픽셀아트 룩. AA 제거, RenderTexture 다운샘플링(Point 필터링), Depth+Normal 외곽선(Fullscreen Shader Graph). 2D-HD 스타일 지양.
[Heading 2] 10.2 캐릭터 비주얼
[Normal] 스프라이트 1세트. 속성 차이는 이펙트 레이어로만 표현: 버프 VFX + 히트 이펙트 색상 + 스킬 연출. 셸이더 파라미터 + 파티클 프리팭 SO로 관리.
[Heading 2] 10.3 오디오
[Normal] 전기 히어로 전용: 스타일 랭크에 연동된 BGM 레이어링. 다른 히어로는 고정 BGM + SFX 강도 변화로 처리.

[Heading 1] 11. 개발 로드맵

[Normal] 목표: 1인 개발, ~10-12개월 → Steam EA 출시 → 퍼블리셔/기업 컨택.

[TABLE_START]
페이즈 | 기간 | 산출물
Phase 0 코어 검증 | 1~2개월 | 픽셀 셸이더 파이프라인 (3D→2D) / 캐릭터 전투 구조 (자동 워핑, 탭/홀드, FSM, 히트 리액션) / 그룹 토큰 AI + 게임필 레이어 / 결과: 잡몹 5명 1스테이지에서 프리플로우 재미 판정
Phase 1 버티컬 슬라이스 | 2~3개월 | 적 4타입 + 환경 오브젝트 / 스타일 랭크 + 연출 에스컬레이션 / 배경 1세트 + 보스 1체 / 3~4스테이지 플레이어블 데모 + SNS 공개 시작
Phase 2 SNS 바이럴 | 병행 | 클립용 GIF 캡처: 벽 파괴, S랭크 처형, 연쇄 폭발 / 트위터/레딧/틱톡 개발로그 / 테스트: GIF 3장으로 게임이 설명되는가?
Phase 3 넥스트 페스트 데모 | 2~3개월 | 히어로 1종(전기) + 패시브 5~8개 / 로그라이크 루프 (스테이지 선택 → 전투 → 보상 → 반복) / BGM 레이어링 + 쉘터 기본 / 목표: 위시리스트 5,000개 이상
Phase 4 EA 빌드 | 3~4개월 | 히어로 3종 완성 (전기/파이어/아이스) / 적 4타입 × 3팔레트 + 배경 3세트 / 패시브 15~20개 + 리스크-리워드 / 영구 해금 스킬 2~3개 + 보스 3체 + 어시스트 3종 / 스토리 도입부. 플레이타임 3~5시간.
Phase 5 EA 출시 | 출시 | 스팀 EA 출시 + 공개 로드맵 / 퍼블리셔 피칭: 트레일러 + 원페이저 + 지표 / 목표: 위시리스트 1만+ 또는 바이럴 GIF 10만뷰+
[TABLE_END]
