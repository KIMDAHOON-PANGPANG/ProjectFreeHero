## 2. 공격

- [x] **2.1 HealthComponent** — `Script/Core/HealthComponent.as` — `class UHealthComponent : UActorComponent` 상속. UPROPERTY `MaxHealth=100.0f`, `CurrentHealth=100.0f`. UFUNCTION(BlueprintCallable) `ApplyDamage(float)` / `Heal(float)` / `IsDead()` / `IsExecutable()` (HP <= MaxHealth * 0.2). 음수/과다 값은 클램프. Log 출력. 검증: `compile_blueprint` 후 BP 우클릭 메뉴에 4개 함수 노출.

- [x] **2.2 AttackComponent** — `Script/Gameplay/AttackComponent.as` — `class UAttackComponent : UActorComponent`. UFUNCTION(BlueprintCallable) `Attack()` — 오너 전방 300uu 구 트레이스(r=80), 히트 Actor 에서 `UHealthComponent` 찾아 `ApplyDamage(20)`. 같은 프레임 중복 히트 차단. 검증: BP 에서 함수 호출 → 로그 `Hit: <actor> -20 HP`.

- [x] **2.3 적 BP + 배치** — `BP_SideScrolling_NPC` 에 `UHealthComponent` 추가 (MCP `component` action=`add_component`), `MaxHealth=60`. `Lvl_SideScrolling` 에 3개 배치 (MCP `level_actor` action=`spawn_actor`, 플레이어 주변 반경 500uu). 검증: Play 중 더미 3개 보이고 Details 에 Health 섹션 노출.

- [x] **2.4 플레이어 공격 입력** — 기존 `IA_ComboAttack` (Variant_Combat 자산) 재활용. `IMC_SideScroller` 에 Left Mouse Button + Gamepad LeftShoulder(LB) 매핑 (Pressed). `BP_SideScrollingCharacter` (= PlayerCharacter) 에 `UAttackComponent` 부착 + `EnhancedInputAction IA_ComboAttack` Triggered → `AttackComponent.Attack()` 바인딩. 검증: 2.2/2.3 Play 로그에서 LB/LMB 입력으로 NPC HP 감소 + 인스턴스별 30/80/150 다른 HP 시작 확인 완료. (원안의 `IA_Attack` 신설은 Combat Variant 자산과 중복되어 재활용. FaceButton Bottom 은 IA_Jump 이미 사용 중이라 LeftShoulder 로 대체.)

## 3. 텔레그래프 → 가드 → 반격

- [x] **3.0 간이 GDD 작성** — `design/gdd/telegraph-system.md` — 5섹션 (Overview / Rules / Formulas / Tuning / AC). **의식 모드 임시 전환** — 사용자와 선택지 → 결정 → 초안 → 승인. 핵심 결정: 텔레그래프 시간(기본 0.8s), 가드 완벽 윈도우(0.15s), UI 방식(머리 위 아이콘 vs 바닥 링), 반격 데미지 배율(x1.5). 승인 후 바이브 모드 복귀.

- [ ] **3.1 TelegraphComponent** — `Script/Gameplay/TelegraphComponent.as` — UActorComponent. UFUNCTION `StartTelegraph(float Duration)` / `CancelTelegraph()`. 델리게이트 `OnTelegraphStart`, `OnTelegraphFire`, `OnTelegraphCancelled`. 내부 타이머. 검증: BP 에서 `StartTelegraph(0.8)` 호출 → 0.8s 후 Fire 델리게이트 1회 발행 로그.

- [ ] **3.2 간이 텔레그래프 위젯** — `Content/UI/WBP_TelegraphIndicator` (Widget Blueprint). 머리 위 빨간 ! 아이콘. `TelegraphComponent.OnTelegraphStart` → Show, `OnTelegraphFire`/`OnTelegraphCancelled` → Hide. 적 BP `WidgetComponent` 로 부착. 검증: `StartTelegraph(1.0)` 호출 → 1초간 머리 위 아이콘 보이고 사라짐.

- [ ] **3.3 GuardComponent** — `Script/Gameplay/GuardComponent.as` — UActorComponent. UFUNCTION `StartGuard()` / `EndGuard()`. 내부 상태 `bIsGuarding`, `GuardStartTime`. 적 `TelegraphComponent.OnTelegraphFire` 시 플레이어의 `(CurrentTime - GuardStartTime) < PerfectWindow(0.15s)` 이면 `OnPerfectGuard` 델리게이트, 아니면 `OnHitBlocked`. 검증: 로그로 완벽/일반 가드 구분 확인.

- [ ] **3.4 가드 입력 + 반격 연결** — `IA_Guard` (Input Action, Digital) 생성. Right Mouse / Gamepad FaceButton Right 매핑. `BP_PlayerCharacter` 에 `UGuardComponent` 부착 + Started → `StartGuard()`, Completed → `EndGuard()`. `OnPerfectGuard` 이벤트에 `AttackComponent.Attack()` x1.5 배율 (2.2 의 데미지 상수를 Parameter 로 바꾸거나 별도 `Counterattack()` 추가). 검증: 적 텔레그래프 타이밍에 가드 → 반격으로 적 HP 급감 로그.

## 4. 처형

- [ ] **4.1 Executable 감지** — 2.1 의 `HealthComponent.IsExecutable()` (이미 HP <= 20%) 기반. `HealthComponent` 에 델리게이트 `OnBecameExecutable` 추가하고 `ApplyDamage()` 후 임계치 진입 시 1회 브로드캐스트. 검증: 적 HP 를 20 이하로 깎으면 로그 `<actor> is executable`.

- [ ] **4.2 처형 입력 + 액션** — `IA_Execute` (Input Action) 생성. `F` 키 / Gamepad FaceButton Top 매핑. `BP_PlayerCharacter` 에서 Triggered 시 전방 300uu 내 `IsExecutable==true` 적 감지 (공격과 동일 트레이스), 해당 적 `HealthComponent.ApplyDamage(9999)` + 간이 이펙트 (기본 `NS_Explosion` 또는 Print String). 검증: Executable 적에 F 키 → 즉사 + 이펙트.

## 5. 토큰형 그룹 AI

- [ ] **5.0 간이 GDD 작성** — `design/gdd/group-token-ai.md` — 5섹션 (Overview / Rules / Formulas / Tuning / AC). **의식 모드 임시 전환**. 핵심 결정: 토큰 수(기본 N=1), 토큰 획득 거리 조건, 포위 슬롯 수(기본 6), 포위 반경(기본 400uu), 슬롯 할당 알고리즘(가장 가까운 슬롯 vs 각도 균등 분배), 토큰 반환 조건(공격 완료/피격/사망/이탈). 승인 후 바이브 모드 복귀.

- [ ] **5.1 GroupTokenManager** — `Script/Gameplay/GroupTokenManager.as` — `UObject` (WorldSubsystem 또는 단일 인스턴스 Actor). UFUNCTION `RequestToken(UAIController)` → bool, `ReturnToken(UAIController)`. 내부 `MaxTokens=1`, `ActiveHolders` 배열. 검증: BP 에서 2개 컨트롤러가 `RequestToken` 호출 → 첫번째 true, 두번째 false 로그.

- [ ] **5.2 포위 슬롯 계산** — `GroupTokenManager` 에 `GetFlankSlot(APawn Target, int SlotIndex, int TotalSlots)` → FVector. `Target.ActorLocation + FlankRadius * (각도 = SlotIndex * 360/TotalSlots)`. 2D 평면(Y축 고정)이므로 X-Z 평면 원. 검증: 6개 슬롯 GetFlankSlot 출력 좌표가 플레이어 주변 균등 원형.

- [ ] **5.3 AI 컨트롤러 로직** — `Content/Variant_SideScroller/Blueprints/AI/` 의 `AIController_SideScrolling` (또는 신규 `AIC_Grunt`) 에서 주기적(0.5s Timer) `GroupTokenManager.RequestToken(self)` 시도. 토큰 보유 → 플레이어 접근 + 텔레그래프 + 공격 → 완료 시 `ReturnToken`. 미보유 → 빈 슬롯 1개 할당받아 `MoveTo(GetFlankSlot)`. StateTree 2상태(`AttackState`, `FlankState`). 검증: 단독 테스트 — 적 컨트롤러 하나로 토큰 요청·공격·반환 사이클 로그.

- [ ] **5.4 포위 AI 통합 시연** — `Lvl_SideScrolling` 에 `BP_Enemy_Grunt` 5개 배치 (MCP `level_actor`). Play 진입 시 5명 중 1명만 공격 접근, 4명은 원형 포위 대기. 공격자 데미지/사망 시 다른 적이 토큰 이어받음. 검증 (핵심 시연): **3분 Play 중 동시 공격자 1명 고정**, 사망·피격 시 교체, 나머지 포위 유지.

## 실행 규칙 (Claude 읽기용)

1. 사용자가 `"다음"` / `"M1 진행"` / `"continue"` / `"M1 next"` 라고 하면 본 파일을 읽고 **0 → 5 섹션 순서로 첫 `- [ ]` 항목** 을 식별.
2. 해당 항목의 inline 명세대로 실행 (바이브 모드 기본). GDD 선행 항목(3.0, 5.0) 진입 시 사용자에게 "의식 모드 임시 전환합니다" 알림 후 선택지 → 결정 → 초안 → 승인 4단계 수행.
3. 검증 통과 시:
   - `Edit` 도구로 해당 `- [ ]` 를 `- [x]` 로 변경 (**삭제 금지 — 히스토리 유지**)
   - 변경된 파일들과 M1.md 를 함께 커밋: `git commit -m "M1: <항목 번호 + 한 줄 요약>"`
   - 사용자에게 바이브 보고 포맷으로 1문장 완료 보고
4. 실패/막힘:
   - 사용자에게 3줄 보고 (뭘 시도했고 / 어디 막혔고 / 뭐 필요한지)
   - 해당 항목은 그대로 둠. 다음 트리거 대기.
5. 사용자가 항목 내용을 수정하고 싶으면 `"0.2 항목 이렇게 바꿔"` 식으로 말하면 Edit 으로 반영.
