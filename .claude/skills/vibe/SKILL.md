## 이 스킬이 하는 일

**이 세션 남은 시간 동안 바이브코딩 규칙을 적용**합니다. 바이브코딩 모드에서:

1. CCGS 5단계 의식 생략 — 바로 코드
2. 서브에이전트(`ue-blueprint-specialist` 등) 호출 금지 — 사용자가 명시 요청하지 않는 한
3. 슬래시 커맨드 체인 금지 — `/gdd-create`, `/review-all-gdds`, `/consistency-check` 등 무거운 스킬 자동 실행 금지
4. "초안 보여드릴까요?" 같은 중간 확인 생략 — 파일 경로·크기가 명시되어 있으면 바로 작성·편집
5. 3옵션 분기형 질문 생략 — 최선의 하나 골라서 실행하고 "이유는 X, 맘에 안 들면 말씀" 형태로 보고

**여전히 지키는 것** (안전장치):
- 파일 수정 전 해당 파일 읽기 (Edit 도구 요구사항)
- 대규모 리팩터·파괴적 git 작업(`reset --hard`, `push -f`) 실행 전 확인
- 에디터·플러그인 재시작 필요한 변경(`.uproject`, `.uplugin`, `Config/`) 전 사전 공지
- `.as` 컴파일 에러 또는 BP compile 에러 나오면 사용자에게 알리고 정지

## Phase 1: 인자 파싱

`$ARGUMENTS`가 비어있으면 → 모드만 활성화하고 "다음 요청 기다리는 중" 상태로 진입.
`$ARGUMENTS`에 태스크가 있으면 → 모드 활성화 후 바로 Phase 3로.

---

## Phase 2: 요청 공식 리마인드 (사용자 메시지 하나 출력)

사용자에게 이것만 짧게 알림:

```
바이브 모드 활성. 요청할 때 이 형식이 최적:

  [파일 경로] + [작은 동사] + [제약 1개] + [검증법]

예: Script/Core/StaminaComponent.as 만들어.
    UActorComponent 상속. Drain/Regen float 함수.
    음수 들어오면 무시. compile_blueprint 로 0 errors 확인.

CCGS 의식은 안 씁니다. 5분 루프로 갑니다.
```

그리고 사용자 다음 요청 기다리기.

---

## Phase 3: 태스크 실행 (인자가 있으면)

사용자가 준 태스크를 아래 규칙으로 실행:

### 3.1 이해 단계 (속으로만, 출력 안 함)

- 파일 경로 — 어디에?
- 책임 — 뭘 하는 클래스·함수?
- 제약 — 값 범위·엣지케이스·예외?
- 검증 — 컴파일? Play 로그? 둘 다?

모호한 항목이 **2개 이상**이면 짧게 되물음. 1개 이하면 합리적 기본값으로 채워서 바로 진행.

### 3.2 실행

- `.as` 작성이면 → `Write` 로 파일 생성·`Edit` 로 수정. 하나의 파일에 40줄 넘으면 분리 권고.
- BP 조작이면 → MCP 도구(`mcp__unreal-handshake__blueprint` 등) 사용.
- 여러 파일 얽히면 → 의존 순서대로 작성·컴파일·다음.

매 단계 끝에 짧은 진행 보고 (1문장).

### 3.3 검증 자동 수행

실행 후 자동으로:
1. `.as` 변경이면 Output Log 확인 (`Saved/Logs/ProjectFreeHero.log` 말미 grep `angelscript.*error`)
2. BP 조작이면 `mcp__unreal-handshake__blueprint(action="compile_blueprint")` 호출
3. 에러 0개면 완료 보고. 1개 이상이면 고쳐서 다시 시도 (최대 3회).

### 3.4 보고 포맷

작업 완료 후 사용자에게 이 형식으로만 보고:

```
✅ 완료: [한 줄 요약]
📁 변경: [파일 목록 — 상대경로]
🔍 검증: [compile 0 errors / Output Log 깨끗 / 기타]
🎯 다음에 해볼 것 (선택): [3개 이하]
```

## Phase 4: 세션 동안 유지할 규칙

이 세션 동안:

**자동 실행 금지**:
- 서브에이전트 스폰 (사용자 명시 요청 시만)
- 무거운 스킬 체인 (`/gdd-*`, `/review-*`, `/consistency-check`, `/gate-check`)
- 메모리 시스템 자동 기록 (사용자 "기억해"라고 하면 그때만)
- 테스트 프레임워크·CI·문서 갱신 작업 (요청되지 않은 이상)

**자동 실행 권장**:
- 파일 작성 후 바로 컴파일 검증
- Hot-reload 결과 확인 (로그 읽기)
- 의도 파악 어려우면 1문장 질문 (3문단 질문 금지)

**요청 해석**:
- "전투 시스템 만들어" → 범위 모호 → "첫 컴포넌트 하나만 만들게요. 예: HealthComponent. OK?"
- "버그 있어" → 맥락 부족 → "파일+줄+재현법 알려주세요"
- "A 또는 B?" → 하나 골라서 시작 — 틀리면 갈아엎을 준비
- 명시적 "승인·리뷰 받아" 단어 있으면 → 그때만 의식 모드로 일시 전환

## Phase 5: 바이브 모드 해제 신호

다음 신호 중 하나라도 감지되면 사용자에게 **"의식 모드로 전환 권고합니다"** 알림:

- 같은 버그가 두 번 이상 재발
- `.as` 파일 개수 10개 초과
- 사용자가 "팀원 합류", "플레이테스트", "출시", "상업" 등 단어 사용
- 한 파일이 150줄 초과
- 단일 태스크가 5번 이상 재작업됨

알림 후에도 사용자가 "계속 바이브로"라고 하면 계속 진행.
