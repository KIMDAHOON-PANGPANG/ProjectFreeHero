# 컨텍스트 관리

컨텍스트는 Claude Code 세션에서 가장 중요한 자원입니다. 적극적으로 관리하세요.

## 파일 기반 상태 (주 전략)

**메모리는 파일이지, 대화가 아닙니다.** 대화는 일시적이며 압축되거나 손실됩니다.
디스크의 파일은 압축과 세션 크래시를 거쳐도 지속됩니다.

### 세션 상태 파일

`production/session-state/active.md` 를 살아있는 체크포인트로 유지합니다.
각 중요한 마일스톤 후에 갱신하세요:

- 디자인 섹션이 승인되고 파일에 기록됨
- 아키텍처 결정이 내려짐
- 구현 마일스톤 도달
- 테스트 결과 획득

상태 파일에는 다음이 포함되어야 합니다: 현재 태스크, 진행 체크리스트,
내려진 핵심 결정, 작업 중인 파일, 미해결 질문.

### 스테이터스 라인 블록 (Production+ 에서만)

프로젝트가 Production, Polish, Release 단계에 있을 때, 스테이터스 라인
스크립트가 파싱할 수 있도록 `active.md` 에 구조화된 상태 블록을 포함하세요:

```markdown
<!-- STATUS -->
Epic: Combat System
Feature: Melee Combat
Task: Implement hitbox detection
<!-- /STATUS -->
```

- 세 필드 (Epic, Feature, Task) 는 모두 선택 — 해당되는 것만 포함
- 포커스 영역을 전환할 때 이 블록을 갱신
- 스테이터스 라인은 이를 브레드크럼으로 표시: `Combat System > Melee Combat > Hitboxes`
- 활성 작업 포커스가 없으면 블록을 제거하거나 비우세요

모든 중단 후 (압축, 크래시, `/clear`), 먼저 상태 파일을 읽으세요.

### 점진적 파일 작성

다중 섹션 문서 (디자인 문서, 아키텍처 문서, 로어 항목) 작성 시:

1. 즉시 스켈레톤으로 파일 생성 (모든 섹션 헤더, 빈 본문)
2. 대화에서 한 번에 한 섹션씩 논의하고 초안
3. 승인되면 즉시 각 섹션을 파일에 기록
4. 각 섹션 후에 세션 상태 파일 갱신
5. 섹션을 기록한 후에는, 그 섹션에 대한 이전 논의가 안전하게 압축될 수 있음
   — 결정은 파일에 있음

이렇게 하면 컨텍스트 윈도우에는 전체 문서 대화 이력(~30-50k 토큰) 대신
*현재* 섹션 논의(~3-5k 토큰)만 보유됩니다.

## 사전 대응적 압축

- **사전 대응적으로 압축**: 한계에서 반응적으로 하지 말고 컨텍스트 사용 ~60-70% 에서 압축
- **`/clear` 사용**: 관련 없는 태스크 사이, 또는 2번 이상 수정 시도 실패 후
- **자연스러운 압축 지점**: 섹션을 파일에 기록한 후, 커밋 후, 태스크 완료 후,
  새 주제 시작 전
- **포커스 압축**: `/compact Focus on [current task] — sections 1-3 are
  written to file, working on section 4`

## 태스크 타입별 컨텍스트 예산

- 가벼움 (읽기/리뷰): 시작 시 ~3k 토큰
- 중간 (기능 구현): ~8k 토큰
- 무거움 (다중 시스템 리팩터): ~15k 토큰

## 서브에이전트 위임

메인 세션을 깨끗하게 유지하기 위해 리서치와 탐색에 서브에이전트를 사용하세요.
서브에이전트는 자체 컨텍스트 윈도우에서 실행되며 요약만 반환합니다:

- **서브에이전트 사용**: 여러 파일에 걸친 조사, 낯선 코드 탐색,
  5k 토큰 이상의 파일 읽기가 필요한 리서치
- **직접 읽기 사용**: 정확히 어떤 1-2 파일을 확인할지 아는 경우
- 서브에이전트는 대화 이력을 상속받지 않음 — 프롬프트에 전체 컨텍스트를 제공하세요

## 압축 지시사항

컨텍스트가 압축될 때 요약에 다음을 보존하세요:

- `production/session-state/active.md` 에 대한 참조 (상태 복구를 위해 읽기)
- 이 세션에서 수정된 파일 목록과 그 목적
- 내려진 아키텍처 결정과 그 근거
- 활성 스프린트 태스크와 현재 상태
- 에이전트 호출과 결과 (성공/실패/블록)
- 테스트 결과 (통과/실패 수, 특정 실패)
- 미해결 블로커 또는 사용자 입력 대기 질문
- 현재 태스크와 어느 단계에 있는지
- 현재 문서의 어느 섹션이 파일에 기록되었고 어느 섹션이 진행 중인지

**압축 후**: `production/session-state/active.md` 와 활발히 작업 중인 파일을 읽어
전체 컨텍스트를 복구하세요. 파일에 결정이 있고, 대화 이력은 부차적입니다.

## 세션 크래시 후 복구

세션이 죽거나 ("prompt too long") 작업을 계속하기 위해 새 세션을 시작하는 경우:

1. `session-start.sh` 훅이 자동으로 `active.md` 를 감지하고 미리보기
2. 컨텍스트를 위해 전체 상태 파일 읽기
3. 상태 파일에 나열된 부분 완료된 파일 읽기
4. 다음 미완료 섹션 또는 태스크부터 계속
